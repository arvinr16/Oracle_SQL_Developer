=> Singkatan:
1. PK    = Primary Key
2. FK    = Foreign Key

=> Persamaan antara Oracle SQL Data Modeler dan Oracle SQL Developer
1. Entitas  =   Tabel
   >Oracle SQL Data Modeler
    Entitas adalah objek konsep yang mewakili sesuatu di dunia nyata.
    Contoh: Pelanggan, Mobil, Transaksi, dll.
   >Oracle SQL Developer
    Entitas akan berubah menjadi Tabel, Nama entitas = nama tabel.

2. Atribut  =   Kolom (Field)
   >Oracle SQL Data Modeler
    Atribut adalah data yang dimiliki oleh suatu entitas.
    Contoh: ID_Pelanggan, Nama, Alamat, dll.
   >Oracle SQL Developer
    Atribut akan berubah menjadi kolom di dalam tabel.

=> Perbedaan antara Oracle SQL Data Modeler dan Oracle SQL Developer
1. Foreign Key:
   >Oracle SQL Data Modeler
    Tidak perlu buat manual di tabel/entitas kolom untuk menyimpan Foreign Key.
    1. Jangan buat attribut FK secara manual.
    2. Buat relasi antar entitas.
    3. Biarkan Data Modeler yang membuat FK nanti di Physical Model.
   >Oracle SQL Developer
    Wajib membuat kolom/attribut secara manual untuk menyimpan Foreign Key.
    1. Wajib punya kolom FK.
    2. Baru bisa mempunyai Foreign Key.
